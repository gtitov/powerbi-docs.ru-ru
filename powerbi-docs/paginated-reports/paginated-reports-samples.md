# <a name="sample-power-bi-paginated-reports"></a>Примеры отчетов Power BI с разбивкой на страницы


[!INCLUDE [applies-to](../includes/applies-to.md)] [!INCLUDE [yes-service](../includes/yes-service.md)] [!INCLUDE [yes-paginated](../includes/yes-paginated.md)] [!INCLUDE [yes-premium](../includes/yes-premium.md)] [!INCLUDE [no-desktop](../includes/no-desktop.md)]

Эта статья содержит сведения и ссылки для нескольких примеров отчетов Power BI с разбивкой на страницы, которые можно скачать и изучить. Они демонстрируют типичные способы использования отчетов с разбивкой на страницы.

## <a name="prerequisites"></a>Предварительные требования

- Эти отчеты можно совместно использовать в сети в исходном виде — без редактирования. Для этого необходима лицензия Power BI Pro. Вы можете зарегистрироваться для получения [бесплатной пробной лицензии на Power BI Pro](../fundamentals/service-self-service-signup-for-power-bi.md#sign-up-for-an-individual-trial-of-power-bi-pro).
- Также необходим доступ к рабочей области Power BI в емкости [Premium](../admin/service-premium-what-is.md).
- Чтобы изменить эти отчеты, необходимо [установить Power BI Report Builder](https://aka.ms/pbireportbuilder) из Центра загрузки Майкрософт.
- Итак, вы готовы скачать эти примеры отчетов с разбивкой на страницы из GitHub. Учетная запись GitHub не требуется. 

## <a name="download-the-reports"></a>Загрузка отчетов

Для успешного скачивания отчетов необходимо скачать репозиторий в виде ZIP-файла, а затем извлечь его. Отчеты с разбивкой на страницы это RDL-файлы.

1. Откройте [репозиторий GitHub Reporting Services](https://github.com/microsoft/Reporting-Services).
1. Щелкните стрелку на зеленой кнопке **Код**, затем выберите **Скачать ZIP-файл**.

    :::image type="content" source="media/paginated-reports-samples/paginated-report-download-zip.png" alt-text="Снимок экрана репозитория GitHub, содержащего пример отчетов Power BI с разбивкой на страницы.":::
    
1. Откройте файл, выберите **Извлечь все** и выберите расположение для файлов. По умолчанию используется имя папки **Reporting-Services-master**.
1. Откройте папку **Reporting-Services-master**, а затем — папку **PaginatedReportSamples**.

    >[!NOTE]
    >Вы можете удалить все остальные папки в папке **Reporting-Services-master**. Они содержат другие примеры, которые вам не нужны.

1. Выберите один из RDL-файлов, чтобы открыть его в Power BI Report Builder.
1. Теперь вы можете [опубликовать отчет с разбивкой на страницы в службу Power BI](paginated-reports-save-to-power-bi-service.md).

## <a name="invoice"></a>Счет

:::image type="content" source="media/paginated-reports-samples/paginated-report-invoice.png" alt-text="Снимок экрана: пример отчета Power BI с разбивкой на страницы для счета":::


Этот отчет с разбивкой на страницы является автономным счетом. Сценарий для этого отчета заключается в том, что вам нужен счет, который можно распечатать с точностью до пикселя. Он должен показывать сведения об общем объеме продаж с указанием описаний позиций, количества, скидок и затрат.

В этом примере показаны уникальные характеристики для создания реальных счетов, например следующие.  

- Табликс (область данных, которая является базовой как для таблиц, так и для матриц). В ней отображается динамически создаваемое содержимое, определяемое пользователем, а также тема.
- Прямоугольная область данных, помещенная в каждую строку табликса текста отчета.
- Элементы отчета, такие как текстовые поля и линии.
- Параметр отчета для динамического выбора содержимого. Отображаемое содержимое применяется к определенной теме посредством заполнителей выражения. 

Источник данных: включается в RDL-файл.

## <a name="labels"></a>Метки

:::image type="content" source="media/paginated-reports-samples/paginated-report-labels.png" alt-text="Снимок экрана: метки отчета с разбивкой на страницы":::

Это пример автономного отчета с разбивкой на страницы. Отчет с несколькими столбцами идеально подходит для разметки шаблона почтовой наклейки. 

Отчеты с метками просты, но обладают несколькими уникальными характеристиками для создания этикетки с разбивкой на страницы.

- Табликс с фиксированным числом столбцов, равным трем, с определенными промежутками между столбцами.
- Прямоугольная область данных, которая повторяется в строках и столбцах на печатной странице.
- Параметр отчета для выбора содержимого, отображаемого в каждой прямоугольной области данных.

Источник данных: включается в RDL-файл.

- Прямоугольная область данных, размещаемая в разных разделах текста отчета.

## <a name="sales-performance"></a>Эффективность продаж

:::image type="content" source="media/paginated-reports-samples/paginated-report-sales-performance.png" alt-text="Снимок экрана: пример отчета Power BI с разбивкой на страницы для показателей продаж":::

Показатели продаж по стране — это пример автономного отчета с разбивкой на страницы. Сценарий для этого отчета заключается в том, что вам нужен счет, который можно распечатать с точностью до пикселя, для просмотра общего объема продаж и других сведений. В нем демонстрируются следующие функции.

- Использование параметра для развертывания сведений в таблице.
- Верхние и нижние колонтитулы.
- Элементы отчета, такие как текстовые поля, линии и прямоугольники, с использованием заполнителей выражений.
- Гистограммы.
- Линии трендов.
- Панели датчиков.

Источник данных: включается в RDL-файл.
