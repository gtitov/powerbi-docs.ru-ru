# <a name="create-a-link-to-a-specific-location-in-the-power-bi-mobile-apps"></a>Создание ссылки на определенное расположение в мобильных приложениях Power BI
Вы можете использовать ссылки для прямого доступа к конкретному содержимому Power BI, например к конкретному отчету, странице отчета, панели мониторинга, плитке и т. д.

В большинстве случаев доступно два сценария использования ссылок для доступа к содержимому в мобильных приложениях Power BI: 

* Чтобы открыть Power BI **за пределами мобильного приложения** и отобразить определенное содержимое. Обычно это сценарий интеграции, в котором вы открываете мобильное приложение Power BI из другого приложения. 
* Чтобы **перейти** внутрь Power BI. Обычно это имеет место, когда требуется создать пользовательскую навигацию в Power BI.

В этой статье рассматриваются следующие темы:
* Использование ссылок для открытия конкретного содержимого Power BI за пределами мобильного приложения. Описано два формата ссылок. В одном из них используется метод перенаправления, и его можно использовать независимо от того, где будет открыт Power BI. Другой открывается непосредственно в мобильном приложении Power BI и будет работать только на мобильных устройствах, на которых установлено мобильное приложение.
* Использование ссылок внутри Power BI для перехода к конкретному содержимому Power BI.

## <a name="use-links-from-outside-the-mobile-app"></a>Использование ссылок за пределами мобильного приложения
Если вы хотите создать ссылку на конкретный элемент в Power BI за пределами мобильного приложения, есть два варианта в зависимости от места открытия ссылки:

* Если вы хотите, чтобы ссылка открывалась правильно независимо от того, была она нажата в браузере на компьютере или на мобильном устройстве, можно создать ссылку, которая будет гарантированно открываться правильно при переходе из любого места. Эта ссылка имеет специальный синтаксис перенаправления для включения такого интеллектуального поведения.

* Если вы уверены, что ссылка будет открываться только на мобильном устройстве с установленным мобильным приложением Power BI, можно избежать издержек перенаправления описанного выше метода и использовать другой синтаксис ссылок, который открывает ссылку непосредственно в мобильном приложении Power BI на мобильном устройстве. Важно отметить, что, хотя эта ссылка позволяет избежать издержек перенаправления, как в первом методе, она будет открываться только на мобильном устройстве с установленным мобильным приложением Power BI.

### <a name="create-a-link-that-works-anywhere"></a>Создание ссылки, которая работает где угодно
Формат ссылки, описанный в этом разделе, использует перенаправление, чтобы ссылка открывалась из любого места.
* Если щелкнуть ссылку на мобильном устройстве, она откроется в мобильном приложении Power BI. Если мобильное приложение на устройстве не установлено, пользователю будет предложено загрузить его в магазине.
* Если щелкнуть ссылку на компьютере, откроется соответствующий элемент на веб-портале Power BI.

Ссылка должна начинаться со специального префикса, за которым следуют параметры запроса:

https<nolink>://app.powerbi.com/Redirect? **[ПАРАМЕТРЫ ЗАПРОСА]** </code>

> [!IMPORTANT]
> Если содержимое размещено в специальном центре обработки данных, например для государственных учреждений или Китая, ссылка должна начинаться с соответствующего адреса Power BI, например **app.powerbigov.us** или **app.powerbi.cn**.

Параметры запроса:

|Параметр  | Значение  | Описание |
|---------|---------|---------|
|**action** (обязательно)    | OpenApp<br>OpenReport<br>OpenDashboard<br>OpenTile | |
|**appId**| Идентификатор GUID из 36 символов | Необходимо указать, если нужно открыть отчет или панель мониторинга, являющиеся частью приложения.<br>Пример: **appId=baf4b16d-b5bd-4360-8a3a-51d11242c09b** |
|**groupObjectId**| Идентификатор GUID из 36 символов | Указывает рабочую область, если нужно открыть отчет или панель мониторинга, не являющиеся частью раздела "Моя рабочая область".<br>Пример: **groupObjectId=9a3841c6-74b3-46f1-85fd-bdd78f27b30e** |
| **dashboardObjectId** | Идентификатор GUID из 36 символов | Идентификатор объекта панели мониторинга (если для параметра action задано значение OpenDashboard или OpenTile)<br>Пример: **dashboardObjectId=033bb049-5b68-4392-b3ef-ae9a43738a4a** |
| **reportObjectId** | Идентификатор GUID из 36 символов | Идентификатор объекта отчета (если для параметра action задано значение OpenReport)<br>Пример: **reportObjectId=6114cec7-78e1-4926-88ff-0bc5338452cf** |
| **tileObjectId** | Идентификатор GUID из 36 символов | Идентификатор объекта плитки (если для параметра action задано значение OpenTile)<br>Пример: **tileObjectId=a845dcb8-a289-43a8-94ea-67a8c0a068f9** |
| **reportPage** | ReportSection&lt;num&gt; | Имя страницы, если нужно открыть определенную страницу отчета. (если параметр action имеет значение OpenReport)<br>Пример: **reportPage=ReportSection6**  |
| **bookmarkGuid** | Идентификатор GUID из 36 символов | Идентификатор закладки, если нужно открыть определенное представление с закладками. (если параметр action имеет значение OpenReport)<br>Пример: **bookmarkGuid=18e8872f-6db8-4cf8-8298-3b2ab254cc7f** |
| **ctid** | Идентификатор GUID из 36 символов | Идентификатор организации элемента (относится к сценариям B2B, его можно опустить, если элемент относится к организации пользователя)<br>Пример: **ctid=5367c770-09d0-4110-bf6a-d760cb5ef681**. |
||||

**Примеры**

В следующих примерах заполнители для значений параметров выделены полужирным шрифтом. Чтобы получить фактические значения, перейдите к службе Power BI, откройте элемент, который необходимо связать, и извлеките нужные значения из URL-адреса элемента.

* **Запуск приложения**

    https<nolink>://app.powerbi.com/Redirect?action=OpenApp&appId= **&lt;appid-guid&gt;** &ctid= **&lt;ctid-guid&gt;**
   
* **Открытие панели мониторинга, которая включена в приложение**

    https<nolink>://app.powerbi.com/Redirect?action=OpenDashboard&appId= **&lt;appid-guid&gt;** &dashboardObjectId= **&lt;dashboardid-guid&gt;** &ctid= **&lt;ctid-guid&gt;**

* **Открытие отчета, который является частью рабочей области, кроме раздела "Моя рабочая область"**

    https<nolink>://app.powerbi.com/Redirect?Action=OpenReport&reportObjectId= **&lt;reportid-guid&gt;** &groupObjectId= **&lt;groupobjectid-guid&gt;** &reportPage=**ReportSection&lt;num&gt;**

### <a name="how-to-get-the-correct-link-format"></a>Получение правильного формата ссылки

#### <a name="links-to-apps-and-items-in-apps"></a>Ссылки на приложения и элементы в приложениях

Для **приложений, а также отчетов и панелей мониторинга, являющихся частью приложения**, получить ссылку проще всего, перейдя в рабочую область приложения и выбрав **Обновить приложение**. Откроется интерфейс публикации приложения. Откройте вкладку разрешений и разверните раздел ссылок, чтобы просмотреть ссылки на приложение и все его содержимое. Эти ссылки можно использовать за пределами Power BI для доступа к приложению и его содержимому напрямую.

![Публикация ссылок приложения через Power BI ](./media/mobile-apps-links/mobile-link-copy-app-links.png)

#### <a name="links-to-items-that-are-not-in-an-app"></a>Ссылки на элементы за пределами приложения 

Для отчетов и панелей мониторинга, которые не являются частью приложения, нужно извлечь идентификаторы нужных объектов из URL-адреса элемента. Для этого перейдите к службе Power BI, найдите элемент, на который требуется ссылка, и отыщите нужные значения в URL-адресе, который отображается в адресной строке браузера.

В приведенных ниже примерах показано, где можно найти нужные идентификаторы в URL-адресах элементов.

* Чтобы найти идентификатор объекта панели мониторинга из 36 символов, перейдите к нужной панели мониторинга в службе Power BI и найдите идентификатор объекта панели мониторинга и другие необходимые идентификаторы в указанных ниже местах:

    https<nolink>://app.powerbi.com/groups/me/dashboards/ **&lt;dashboard-object-id&gt;** ?ctid= **&lt;org-object-id&gt;**

* Чтобы найти 36-символьный идентификатор объекта отчета, перейдите к конкретному отчету в службе Power BI и найдите необходимые идентификаторы, как показано ниже. Обратите внимание, что этот пример содержит ссылку на определенную страницу отчета и определенную закладку.

    https<nolink>://app.powerbi.com/groups/me/reports/ **&lt;report-object-id&gt;** /**ReportSection&lt;num&gt;** ?bookmarkGuid= **&lt;bookmark-id&gt;**

* Чтобы создать ссылку на элемент в рабочей области, кроме раздела "Моя рабочая область", необходимо извлечь идентификатор объекта группы. В этом примере показан отчет из рабочей области, кроме раздела "Моя рабочая область".

    https<nolink>://app.powerbi.com/groups/ **&lt;group-object-id&gt;** /reports/ **&lt;report-object-id&gt;** /**ReportSection&lt;report-section-num&gt;** ?ctid= **&lt;org-object-id&gt;**

### <a name="create-a-link-that-opens-only-on-a-device-that-has-the-power-bi-mobile-app-installed"></a>Создание ссылки, которая открывается только на устройстве с установленным мобильным приложением Power BI

Формат ссылки, описанный в этом разделе, связан с конкретным расположением в мобильных приложениях Power BI на всех мобильных платформах: iOS, устройства Android и Windows 10. Этот формат ссылки открывает расположение напрямую, без перенаправления, как в методе, описанном в предыдущем разделе. **Этот формат можно открыть только на мобильных устройствах, на которых установлено мобильное приложение Power BI**.

Ссылки этого формата могут указывать непосредственно на панели мониторинга, плитки и отчеты. Формат определяется назначением прямой ссылки. Чтобы создать прямые ссылки на различные расположения, сделайте следующее. 

* **Открытие мобильного приложения Power BI**

    Используйте эту ссылку, чтобы открыть мобильное приложение Power BI на любом устройстве:

    mspbi://app/

* **Открытие определенной панели мониторинга**

    Эта ссылка открывает определенную панель мониторинга в мобильном приложении Power BI:

    mspbi://app/OpenDashboard?DashboardObjectId= **<36-character-dashboard-id>**

    Чтобы найти 36-символьный идентификатор объекта панели мониторинга, перейдите на необходимую панель мониторинга в службе Power BI и извлеките его из URL-адреса. Например, идентификатор объекта панели мониторинга выделен в следующем URL-адресе из службы Power BI:

    https<nolink>://app.powerbi.com/groups/me/dashboards/ **&lt;61b7e871-cb98-48ed-bddc-6572c921e270&gt;**

    Если панель мониторинга не находится в области "Моя рабочая область", необходимо также добавить идентификатор объекта группы (до или после идентификатора панели мониторинга). Ниже приведена прямая ссылка с идентификатором объекта группы, добавленным после идентификатора объекта панели мониторинга:

    mspbi://app/OpenDashboard?DashboardObjectId=**e684af3a-9e7f-44ee-b679-b9a1c59b5d60**&GroupObjectId=**8cc900cc-7339-467f-8900-fec82d748248**</code>

    Обратите внимание на амперсанд (&) между двумя параметрами.

* **Открытие определенной плитки в режиме фокусировки**

    Эта ссылка открывает определенную плитку в режиме фокусировки в мобильном приложении Power BI:

    mspbi://app/OpenTile?DashboardObjectId= **<36-character-dashboard-id>** &TileObjectId= **<36-character-tile-id>**

    Чтобы найти 36-символьные идентификаторы объектов панели мониторинга и плитки, перейдите на требуемую панель мониторинга в службе Power BI и откройте плитку в режиме фокусировки. В примере ниже выделены идентификаторы панели мониторинга и плитки.

    https<nolink>://app.powerbi.com/groups/me/dashboards/**3784f99f-b460-4d5e-b86c-b6d8f7ec54b7**/tiles/**565f9740-5131-4648-87f2-f79c4cf9c5f5**/infocus

    Ссылка для открытия плитки напрямую будет выглядеть следующим образом:

    mspbi://app/OpenTile?DashboardObjectId=3784f99f-b460-4d5e-b86c-b6d8f7ec54b7&TileObjectId=565f9740-5131-4648-87f2-f79c4cf9c5f5

    Обратите внимание на амперсанд (&) между двумя параметрами.

    Если панель мониторинга не находится в области "Моя рабочая область", добавьте параметр GroupObjectId, например &GroupObjectId=<36-character-group-id>

* **Открытие определенного отчета**

    Эта ссылка открывает определенный отчет в мобильном приложении Power BI:

    mspbi://app/OpenReport?ReportObjectId= **<36-character-report-id>**

    Чтобы найти 36-символьный идентификатор объекта отчета, перейдите к требуемому отчету в службе Power BI. Следующий URL-адрес из службы Power BI показывает идентификатор отчета, который необходимо извлечь.

    https<nolink>://app.powerbi.com/groups/me/reports/**df9f0e94-31df-450b-b97f-4461a7e4d300**

    Если отчет не находится в области "Моя рабочая область", необходимо также добавить **&GroupObjectId=<36-character-group-id>** до или после идентификатора отчета. Например, в этом случае прямая ссылка будет выглядеть следующим образом:

    mspbi://app/OpenReport?ReportObjectId=**e684af3a-9e7f-44ee-b679-b9a1c59b5d60**&GroupObjectId=**8cc900cc-7339-467f-8900-fec82d748248**

    Обратите внимание на амперсанд (&) между двумя параметрами.

* **Открытие определенной страницы отчета**

    Эта ссылка открывает определенную страницу отчета в мобильном приложении Power BI:

    mspbi://app/OpenReport?ReportObjectId= **<36-character-report-id>** &reportPage=**ReportSection&lt;number&gt;**

    Название страницы отчета состоит из текстового блока **ReportSection** и определенного номера. Чтобы найти нужные значения, откройте отчет в службе Power BI, перейдите на конкретную страницу отчета и извлеките значения из URL-адреса. Например, выделенные разделы этого URL-адреса представляют значения, которые необходимы, чтобы открыть конкретную страницу отчета:

    https<nolink>://app.powerbi.com/groups/me/reports/**df9f0e94-31df-450b-b97f-4461a7e4d300**/**ReportSection11**</code>

* **Открытие в полноэкранном режиме (только для устройств Windows)**

    Для устройств Windows можно также добавить параметр **openFullScreen**, чтобы открыть конкретный отчет в полноэкранном режиме. В следующем примере страница отчета открывается в полноэкранном режиме:

    mspbi://app/OpenReport?ReportObjectId=500217de-50f0-4af1-b345-b81027224033&**openFullScreen=true**

* **Добавление контекста** (необязательно)

    Вы также можете добавить в строку контекст. Затем, если вам понадобится связаться с нами, мы используем этот контекст для фильтрации наших данных, чтобы найти связь в вашем приложении. Чтобы добавить контекст, добавьте в ссылку параметр **context=&lt;app-name&gt;** :

    Например, в следующем примере показана ссылка, включающая параметр контекста: 

    mspbi://app/OpenReport?ReportObjectId=**e684af3a-9e7f-44ee-b679-b9a1c59b5d60**&GroupObjectId=**8cc900cc-7339-467f-8900-fec82d748248**&**context=SlackDeepLink**

## <a name="use-links-inside-power-bi"></a>Использование ссылок внутри Power BI

В мобильных приложениях Power BI ссылки внутри Power BI работают точно так же, как в службе Power BI.

Если вы хотите добавить ссылку на отчет, указывающий на другой элемент Power BI, можно просто скопировать URL-адрес этого элемента из адресной строки браузера. См. дополнительные сведения о [добавлении гиперссылки в текстовое поле в отчете](../../create-reports/service-add-hyperlink-to-text-box.md).
