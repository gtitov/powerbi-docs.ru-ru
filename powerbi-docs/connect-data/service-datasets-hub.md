# <a name="datasets-discovery-using-the-datasets-hub"></a>Обнаружение наборов данных с помощью концентратора наборов данных

Концентратор наборов данных упрощает поиск, изучение и использование наборов данных в организации. Он предоставляет сведения о наборах данных, а также точки входа для создания отчетов на основе этих наборов данных или для использования этих наборов данных с помощью функции анализа в Excel.

Концентратор наборов данных может быть полезным во многих сценариях:
* Владельцы наборов данных могут просматривать метрики использования набора данных, состояние обновления, связанные отчеты и журнал происхождения данных для отслеживания своих наборов данных и управления ими.
* Создатели отчетов могут использовать концентратор, чтобы находить подходящие наборы данных для создания отчетов и использовать ссылки для создания отчетов на основе набора данных с нуля или с помощью шаблонов.
* Пользователи отчетов могут использовать эту страницу для поиска отчетов на основе надежных наборов данных.

Упрощая поиск качественных наборов данных и связанных с ними отчетов, концентратор наборов данных помогает предотвратить создание избыточных отчетов. Он также упрощает поиск качественных отчетов для использования в качестве отправных точек при создании новых отчетов.

В этой статье показано, что отображается в концентраторе наборов данных, и описано, как концентратор использовать. Владельцам наборов данных предоставляется ряд советов по [оптимизации возможности обнаружения и удобства использования своих наборов данных](#make-your-dataset-discoverable).

**Какие наборы данных отображаются в концентраторе наборов данных?**
* Чтобы набор данных отображался в концентраторе, он должен находиться в [новой рабочей области](../collaborate-share/service-new-workspaces.md).
* В концентраторе наборов данных для вас будут отображаться те наборы данных, для которых у вас есть хотя бы [разрешения на сборку](service-datasets-build-permissions.md).
* Если вы являетесь пользователем категории "Бесплатный", вы увидите только наборы данных в разделе *Моя рабочая область* или наборы данных, для которых у вас есть [разрешения на сборку](service-datasets-build-permissions.md) и которые находятся в рабочих областях в емкости Premium.

## <a name="find-the-dataset-you-need"></a>Поиск нужного набора данных

Обнаружение набора данных начинается с перехода на страницу концентратора наборов данных. Для этого:
* В службе Power BI: на панели навигации выберите **Наборы данных**.
* В приложении Power BI в Teams: в области навигации выберите **Наборы данных** или перейдите на вкладку **Наборы данных**.

На рисунке ниже показан концентратор наборов данных в службе Power BI.

![Снимок экрана: страница концентратора наборов данных](media/service-datasets-hub/datasets-hub-main-page.png)

Концентратор наборов данных представляет ряд рекомендуемых наборов данных и список всех наборов данных в организации, на доступ к которым у вас есть разрешения.

В следующих разделах описываются эти наборы данных, а также действия, которые можно выполнить.

### <a name="recommended-datasets"></a>Рекомендуемые наборы данных

Рекомендуемые наборы данных — это подтвержденные наборы данных (предлагаемые или сертифицированные), которые предоставляются на основе расчетов, в которых учитывается, как давно они были обновлены и как давно вы посещали отчеты и (или) панели мониторинга, связанные с ними.

### <a name="dataset-list"></a>Список наборов данных

В списке наборов данных отображаются наборы данных в организации, для которых у вас есть хотя бы [разрешения на сборку](service-datasets-build-permissions.md). Список содержит три вкладки для фильтрации списка наборов данных.
* **Все наборы данных.** Здесь отображаются все наборы данных в организации, для которых у вас есть хотя бы [разрешения на сборку](service-datasets-build-permissions.md).
* **Недавние.** Здесь отображаются наборы данных со связанными отчетами, к которым вы недавно обращались. При обращении к отчету может возникнуть задержка в несколько минут, пока связанный набор данных не отобразится в столбце недавних наборов данных.
* **Мои наборы данных.** Здесь отображаются наборы данных, которыми вы владеете. 

Используйте поле поиска для дальнейшей фильтрации элементов на текущей вкладке.

Столбцы списка описаны ниже. Щелкните заголовок столбца для сортировки по этому столбцу. 
* **Имя.** Имя набора данных. Щелкните имя набора данных, чтобы просмотреть отчеты, созданные с его помощью.
* **Подтверждение.** Состояние подтверждения.
* **Владелец.** Владелец набора данных.
* **Рабочая область.** Рабочая область, в которой находится набор данных.
* **Обновлено.** Время последнего обновления. (Отображается час, день, месяц и год. Точное время последнего обновления см. в сведениях о наборе данных на соответствующей странице.)
* **Конфиденциальность.** Метка конфиденциальности (если она задана). Щелкните значок сведений, чтобы просмотреть описание метки конфиденциальности.

### <a name="create-new-reports-or-pull-data-into-excel-via-analyze-in-excel"></a>Создание новых отчетов или извлечение данных в Excel с помощью функции анализа в Excel

Чтобы создать новый отчет на основе набора данных или извлечь данные в Excel с помощью функции [Анализировать в Excel](../collaborate-share/service-analyze-in-excel.md), выберите **Дополнительные параметры (...)** в нижнем правом углу плитки рекомендуемого набора данных или в строке набора данных в списке наборов. В раскрывающемся меню могут отображаться другие действия в зависимости от разрешений, которые назначены вам для набора данных.

При создании нового отчета на основе набора данных открывается холст редактирования отчетов. Новый отчет будет сохранен в рабочей области, содержащей набор данных, если у вас есть разрешения на запись в эту рабочую область. Если у вас нет таких разрешений или если вы являетесь пользователем категории "Бесплатный", а набор данных находится в рабочей области в емкости Premium, новый отчет будет сохранен в разделе *Моя рабочая область*.

## <a name="view-dataset-details-and-explore-related-reports"></a>Просмотр сведений о наборе данных и изучение связанных отчетов

Чтобы просмотреть дополнительные сведения о наборе данных, изучить связанные отчеты или создать новый отчет на основе набора данных с нуля или с помощью шаблона, выберите набор данных из рекомендуемых наборов данных или из списка наборов данных. Откроется страница, на которой отображаются сведения о наборе данных, список отчетов, созданных на основе набора данных, и точки входа для создания новых отчетов на основе набора данных или извлечения данных в Excel с помощью функции [Анализировать в Excel](../collaborate-share/service-analyze-in-excel.md).

![Снимок экрана: страница просмотра связанных отчетов в концентраторе наборов данных](media/service-datasets-hub/datasets-hub-explore-related-reports.png)

В заголовке страницы указаны имя набора данных, подтверждения (если они заданы), а также владелец набора данных. Чтобы отправить сообщение электронной почты владельцу набора данных или лицу, которое сертифицирует наборы данных (если сертификация осуществлялась), щелкните заголовок и выберите имя владельца.

### <a name="dataset-details"></a>Сведения о наборе данных

В разделе сведений о наборе данных отображается имя рабочей области, в которой находится набор данных, точное время последнего обновления, метка конфиденциальности (если она задана), описание набора данных (если оно задано) и имя лица, которое осуществляет сертификацию (если сертификация осуществлялась). Здесь также можно просмотреть сведения о происхождении данных в наборе.

### <a name="related-reports"></a>Связанные отчеты

В разделе просмотра связанных отчетов отображаются все отчеты, созданные на основе выбранного набора данных. Вы можете создать копию отчета, выбрав в списке строку отчета, а затем щелкнув значок сохранения копии этого отчета.

Столбцы в списке связанных отчетов:
* **Имя.** Имя отчета. Если имя оканчивается на "(шаблон)", это означает, что такой отчет специально создан для использования в качестве шаблона.
* **Подтверждение.** Состояние подтверждения.
* **Рабочая область.** Имя рабочей области, в которой находится отчет.

### <a name="create-a-report-built-on-the-dataset"></a>Создание отчета на основе набора данных

В разделе создания отчета нажмите кнопку **Создать**. Если для набора данных доступен шаблон отчета, в раскрывающемся меню будут предложены два варианта:
* **Из шаблона.** Создание копии шаблона в разделе *Моя рабочая область*.
* **С нуля.** Открытие холста редактирования отчета в новом отчете на основе набора данных. Новый отчет будет сохранен в рабочей области, содержащей набор данных, если у вас есть разрешения на запись в эту рабочую область. Если у вас нет таких разрешений или если вы являетесь пользователем категории "Бесплатный", а набор данных находится в рабочей области в емкости Premium, новый отчет будет сохранен в разделе *Моя рабочая область*.

Если шаблоны отчетов отсутствуют, нажав кнопку **Создать**, вы сразу откроете холст редактирования отчета.

>[!NOTE]
> В раскрывающемся списке для создания отчета отобразится только один шаблон, даже если для этого набора данных доступно несколько шаблонов отчета. 

### <a name="pull-the-dataset-into-excel-via-analyze-in-excel"></a>Извлечение набора данных в Excel с помощью функции анализа в Excel

В разделе анализа в Excel щелкните **Анализировать**, чтобы извлечь набор данных в Excel с помощью функции анализа в Excel.

## <a name="make-your-dataset-discoverable"></a>Включение возможности обнаружения набора данных

Есть несколько способов оптимизировать возможность обнаружения наборов данных:
* **Подтвердите набор данных.** Вы можете предложить или сертифицировать набор данных, чтобы пользователи могли легко находить его, зная, что он является надежным источником данных. Подтвержденные наборы данных помечены значками и их легко идентифицировать в Power BI. В концентраторе наборов данных в разделе рекомендуемых наборов данных отображаются только подтвержденные наборы данных. Кроме того, в списке наборов данных подтвержденные наборы данных отображаются первыми по умолчанию.

    [Узнайте, как подтвердить наборы данных.](../collaborate-share/service-endorse-content.md) 
* **Предоставьте содержательное описание набора данных.** Вы можете помочь пользователям быстрее находить нужные им наборы данных, предоставив понятные и информативные описания наборов данных. [Описание предоставляется в рамках процесса подтверждения набора данных.](../collaborate-share/service-endorse-content.md#promote-content) 
* **Присвойте набору данных запоминающееся изображение.** Поиск и выбор набора данных можно упростить, предоставив запоминающееся изображение. Так набор данных будет выделяться на странице концентратора наборов данных и в других местах, где поддерживаются изображения наборов данных. Чтобы присвоить набору данных изображение, откройте параметры набора данных и разверните раздел изображения набора данных.
* **Создайте шаблон отчета на основе набора данных.** Вы можете создать шаблон отчета, с помощью которого пользователи смогут создавать собственные отчеты на основе вашего набора данных. Такой шаблон — это обычный отчет, который предполагается использовать в качестве шаблона. Когда вы будете его сохранять, добавьте "(шаблон)" в конец имени отчета, например *Продажи за месяц (шаблон)* .

    Когда пользователь выберет **Создать > Из шаблона** в разделе создания отчетов в представлении сведений о наборе данных в концентраторе наборов данных, копия шаблона будет создана в разделе *Моя рабочая область*, а затем открыта на холсте редактирования отчетов.

    Шаблоны отчетов также можно легко определить в списке связанных отчетов в представлении сведений о наборе данных в концентраторе наборов данных.
